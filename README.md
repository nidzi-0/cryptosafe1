# CryptoSafe Manager  
## Пояснительная записка к Sprint 1  

---

# 1. Введение

В рамках Sprint 1 проекта «CryptoSafe Manager» был разработан фундамент приложения - архитектурная и техническая основа для последующей реализации функционала безопасного менеджера паролей.

Основной задачей спринта являлось создание:

- модульной архитектуры проекта;
- расширяемой схемы базы данных;
- абстрактного криптографического слоя;
- системы событий;
- графической оболочки;
- инфраструктуры тестирования и CI.

---

# 2. Архитектурные решения

## 2.1 Принцип построения

Проект реализован с использованием принципов разделения ответственности (Separation of Concerns) и модульной архитектуры.

Архитектура приближена к паттерну MVC:

- View — графический интерфейс (модуль gui)
- Model — слой базы данных (database)
- Controller / Business Logic — модуль core

Данная структура позволяет:
- минимизировать связность компонентов,
- обеспечить масштабируемость,
- упростить тестирование,
- заменить криптографическую реализацию без изменения бизнес-логики.
